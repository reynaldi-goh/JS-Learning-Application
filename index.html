Printing

Every program needs a way to talk back to you. In JavaScript, that's done with console.log(). It displays information in the browser console. You can right-click and select "Inspect" or "Developer Tools" to view it.

Developers use it constantly to test ideas, debug code, or simply check if something's working. Think of it as your private notebook where JavaScript whispers what it's doing behind the scenes.

  • Open Developer Tools: Right-click â†’ Inspect â†’ Console tab
  • Use console.log() to display messages
  • Essential for debugging and understanding code flow

Including JavaScript in HTML

There are three ways to include JavaScript: Inline (inside HTML attributes), Internal (inside <script> tags), and External (separate .js file).

  • Inline: Fast for demos, but logic scattered in HTML
  • Internal: Good middle ground for small projects
  • External: Best practice â€” keeps code organized and reusable

This website uses external JavaScript (script.js), which is the professional approach for complex projects.

Variables

Variables are containers that hold data for later use. They let you store information, give it a name, and use it anywhere in your program.

  • let â€” use when the variable might change (like a score or counter)
  • const â€” use when the value must not change (like constants or settings)
  • var â€” outdated, don't use it! It behaves weirdly and let/const are safer

Data Types

JavaScript handles many kinds of data, each suited for different purposes. You can use typeof to check what kind of data something is.

  • String: Text data enclosed in quotes â€” "hello"
  • Number: Numeric values (no separate "int" or "float") â€” 42, 3.14
  • Boolean: Logical values used in decisions â€” true, false
  • Array: Ordered list of items â€” [1, 2, 3]
  • Object: Structured collection of data â€” { name: "Bob", age: 25 }

Conditionals

Conditionals let your code make decisions. They ask questions like "Is this true?" or "Does this number pass the test?" and act accordingly.

  • if, else if, else â€” basic decision making
  • Comparison operators: === (strictly equal), !==, >, <, >=, <=
  • Logical operators: && (AND), || (OR), ! (NOT)
  • Use === instead of == to avoid type coercion surprises

Loops

Loops repeat code automatically â€” so you don't write the same line over and over. There are two main types: for and while loops.

  • For loop: Initialize, test, and update in a single line
  • While loop: Runs while a condition is true
  • Loop through arrays using index: for (let i = 0; i < array.length; i++)
  • Nested loops: loop inside loop for multi-layered data like grids

Instead of writing "Hello 1" through "Hello 1000" manually, loops make it far shorter and more readable!

Functions

Functions are reusable blocks of code designed to perform specific tasks. console.log() is a built-in function â€” you can also create your own custom functions!

  • Define with function functionName() { } and call with functionName()
  • Parameters: placeholders for values you pass in (arguments)
  • Return values to share results with the caller
  • Functions eliminate repetition â€” write once, use many times

Instead of rewriting the same logic for different data, functions let you create reusable patterns. Like a personal calculator bot you can call anytime!

Objects

Objects store data as keyâ€“value pairs â€” a way to represent real-world things with properties. You can access, modify, add, or delete properties.

  • Access: student.name or student['name']
  • Modify: student.age = 20
  • Add: student.city = "Singapore"
